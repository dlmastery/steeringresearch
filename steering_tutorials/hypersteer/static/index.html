HyperSteer  ·  lesson 3 dashboard

Lesson 2 added one fixed refusal direction (a diff-of-means vector) to Gemma-1B's residual stream. Lesson 3 learns a hypernetwork H that generates the direction from a concept: v = H(concept embedding). Once trained it amortises — hand it a brand-new concept's exemplars and it emits a steering vector on the spot, with no fresh extraction pass.
The three-lesson arc:  READ harm (lesson 1) → WRITE a fixed vector (lesson 2) → GENERATE the vector with a hypernet (lesson 3, you are here).

How to read this

  • The vector is generated, not hand-built. When the gate fires we add a direction the hypernetwork emitted from a concept embedding — not lesson-2's fixed diff-of-means. If lesson 2 is also built, you'll see both side by side (learned vs fixed).
  • Amortisation is the payoff. In the second panel, type a few phrases describing any concept and watch the hypernet turn them into a steering vector live — no contrastive extraction, no retraining.
  • Conditioning still holds. Steering every prompt would wreck harmless ones, so we only steer when the lesson-1 gate says a prompt is harmful. Benign prompts pass through untouched.
  • A 1B self-judge is weak. One small model plays writer, gate, and judge. Treat the REFUSAL / COMPLIANCE / GIBBERISH verdicts as illustrative, not publication-grade.
