Conditional steering  ·  lesson 2 dashboard

Lesson 1 READ harm out of a frozen Gemma-1B's activations. Lesson 2 does the WRITE half: it adds a refusal direction back into the residual stream to make an uncensored model refuse. The trick is the conditional gate — steer only when the lesson-1 probe says a prompt is harmful, so benign requests pass through untouched.

How to read this

  • Read → gate → write → judge. Every prompt is first scored by the gate (the lesson-1 probe). Only if it fires do we add the steering vector; then the same 1B model judges whether the result is a REFUSAL, COMPLIANCE, or GIBBERISH.
  • The gate is the point. Steering every prompt would wreck harmless requests. Conditioning keeps the intervention off unless it's needed — watch a benign prompt leave the gate quiet and the output unchanged.
  • Corruption is a real outcome. Push steering too hard and coherent refusals collapse into word-salad. GIBBERISH (amber) is caught first, deterministically, before the refuse-vs-comply call.
  • A 1B self-judge is weak. This is a teaching artifact: one small model plays writer, gate, and judge. Treat verdicts as illustrative, not publication-grade.
